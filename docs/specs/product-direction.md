# 产品方向：时光机

> 版本：v1
> 日期：2026-06-24
> 状态：已确认
>
> 本文档定义产品的核心方向、设计原则和功能体系。所有后续需求讨论和实现决策以此为准。

---

## 产品定位

**一台时光机，而非一个照片容器。**

目标用户是长期用手机积累了大量带有 GPS 信息的照片、想要回顾自己生活轨迹的人。

产品围绕"人在地球上的移动"来组织体验，而非围绕"照片文件"来组织。照片是移动轨迹的证据，不是一等公民。

核心情感价值：**重温记忆**，而非**管理资产**。

---

## 设计原则

1. **移动是一等公民。** 信息的组织和导航围绕用户在地球上的位置变化展开，照片附属于位置和时间。

2. **算法检测，人工修正。** 居住期、旅行、事件由算法自动检测（目标 80% 正确率），用户可手动合并、拆分、命名、忽略。

3. **不修改原图。** 所有用户编辑（命名、收藏、修正）写入 `.locus-metadata.json`，原文件永不修改。

4. **重探索轻管理。** 核心交互是浏览、回放、发现故事。批量操作、搜索筛选等管理功能降级到次级入口。

5. **内容为王。** 地图、轨迹、地名、故事是主角。UI 是安静的容器，不喧宾夺主。

## 核心概念

```
生活轨迹
  ├── 居住期
  │    ├── 旅行（在居住期内离开日常范围的出行）
  │    ├── 事件（在家附近但照片密度异常高的日子）
  │    └── 日常（折叠，不主动展示）
  │
  ├── 迁移（居住期之间的过渡）
  │
  └── 未分类（算法无法归类的照片，降级到全部照片视图）
```

### 居住期

用户在某个城市/区域持续生活超过 2 个月的时期。由 DBSCAN 空间聚类 + 时态过滤自动检测。用户可手动合并、拆分、命名。

典型示例：天津期 14 个月、北京期 11 个月、南京期 13 个月。

### 旅行

在居住期内，连续离开居住地范围且超过 4 小时、至少包含 3 张照片的出行段。相邻的短间隙旅行段可合并。

### 事件

在居住地范围内，某天照片密度异常高（>20 张），可能是聚会、生日、搬家等。自动标记为"家庭活动"。

### 迁移

两个居住期之间的照片段。如果距离两个居住期都很远 → 可能是独立旅行。

## 信息架构

```
应用入口
│
├── 生命轨迹地图（主视图，默认）
│   ├── 居住期热力图层
│   ├── 旅行轨迹图层（polyline + 地名标注）
│   ├── 照片散点图层（缩略图标记）
│   ├── 地图底图切换（高德/CartoDB/OSM）
│   └── 时间游标（底部可拖拽时间轴）
│
├── 旅行列表
│   ├── 旅行卡片（封面 + 日期 + 地点摘要，按时间倒序）
│   └── 旅行详情
│       ├── 地图（轨迹线 + 地名标注）
│       ├── 按天分组的照片流
│       └── 回放
│
├── 故事
│   自动生成的叙述卡片：
│   ├── 时间跨度故事
│   ├── 空间跨度故事
│   ├── 密度故事
│   ├── 对比故事
│   ├── 发现故事
│   └── 里程碑故事
│
└── 全部照片（降级入口）
    传统的年→月→照片瀑布流
    + 搜索筛选（文件名、日期范围、相机型号）
    + 灯箱浏览
    + 元数据校准
```

## 功能体系

### P0 — 产品身份建立

| 功能 | 说明 |
|------|------|
| 居住期检测引擎 | DBSCAN 聚类 + 60天时态过滤 + 短途出行检测 + 迁移检测 |
| 生命轨迹地图 | 新默认首页，居住期热力 + 旅行轨迹 + 照片散点 + 时间游标 |
| 旅行列表 | 旅行卡片列表 + 点击进入旅行详情（地图轨迹 + 按天照片流） |
| 居住期/旅行手动修正 | 合并、拆分、命名、提升为居住期、降级为旅行、忽略噪点 |

### P1 — 体验深化

| 功能 | 说明 |
|------|------|
| 旅行回放 | 地图飞移动画 + 轨迹逐段绘制 + 照片弹出 + 速度控制 |
| 反向地理编码 | 批量高德逆地理编码 + localStorage 缓存 + 地图地名标注 |
| 按天照片流 | 替代旧月分组，显示每天的地点摘要行 |
| 心标（收藏） | 轻量标记，替代五星星级 |

### P2 — 故事与分享

| 功能 | 说明 |
|------|------|
| 故事引擎 | 六类自动生成故事：时长/空间/密度/对比/发现/里程碑 |
| 旅行日记 HTML 导出 | 自包含单文件，内联地图+轨迹+照片，浏览器即开即用 |
| GPX 导出 | 从单段旅行生成轨迹文件 |

### P3 — 支撑能力

| 功能 | 说明 |
|------|------|
| EXIF 详情面板 | 光圈/快门/ISO/焦距/闪光灯/海拔 |
| 视频支持 | MP4/MOV 识别 + 首帧缩略图 + 内嵌播放 |
| 幻灯片播放 | 旅行详情中自动播放 |
| 快捷键面板 | `?` 键唤出 |
| 移动端响应式 | 触控友好布局 |

---

## 明确不做的

以下功能曾出现在讨论中，基于产品定位明确排除或降级：

| 功能 | 决策 | 原因 |
|------|------|------|
| 多选批量操作 | 不做 | 不是旅行记录者的核心场景 |
| 五星星级 + 色标 | 不做 | 过于管理导向，心标即可 |
| 重复/相似照片检测 | 不做 | 不是痛点 |
| 多文件夹聚合 | 不做 | 居住期自动检测替代了手动组织 |
| 通用 JSON 导出 | 不做 | GPX + HTML 旅行日记更精准 |
| 搜索筛选 | 降级 | 仅在"全部照片"视图中提供 |
| PWA | 降级 | 旅行回顾在大屏体验更好 |

## 技术策略

### 居住期检测参数

```
spatial.clusterEps:     0.5 km      DBSCAN 聚类半径
spatial.minClusterPts:  5           最少照片数才算一个地点
residence.minDuration:  60 days     空间聚类时态跨度阈值
residence.minPhotosPerMonth: 3      月均最少照片数
excursion.maxGapHours:  48          离开居住地后合并相邻出行的间隙
excursion.minPhotos:    3           最少照片数才算一段出行
event.densityThreshold: 20          在家附近单日照片数触发事件标记
```

### 数据存储

- 居住期和旅行：**纯计算属性**，从 `photos[]` 派生，不持久化
- 用户修正（命名、合并、拆分、心标）：写入 `.locus-metadata.json`
- 反向地理编码缓存：`localStorage`，按 `lat,lng` key 去重

### 新增模块

```
src/services/
  residenceDetector.js    ← 居住期检测
  tripDetector.js         ← 旅行检测（在居住期内部）
  reverseGeocoder.js      ← 批量反向地理编码 + 缓存
  storyEngine.js          ← 故事生成
  tripExporter.js         ← GPX / HTML 导出

src/views/
  LifemapView.jsx         ← 生命轨迹地图（新默认首页）
  TripListView.jsx        ← 旅行列表
  TripDetailView.jsx      ← 旅行详情
  StoryView.jsx           ← 故事

src/components/
  TripCard.jsx            ← 旅行卡片
  ReplayController.jsx    ← 回放控制
  TimeCursor.jsx          ← 时间游标
  LayerControl.jsx        ← 图层控制
```
